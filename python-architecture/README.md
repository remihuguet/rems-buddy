# python-architecture

Python backend architecture patterns: hexagonal architecture, DDD, CQRS, and event-driven design.

## Skills

- **hexagonal-layers** -- Five-layer architecture: domain, service, adapters, views, entrypoints
- **domain-driven-design** -- Value objects, entities, aggregates, repositories, domain events, commands
- **cqrs** -- Command/query separation for optimized reads and writes
- **event-driven** -- MessageBus, UnitOfWork, command/event handler patterns
- **architecture-pitfalls** -- Common anti-patterns to avoid in layered applications
- **file-organization** -- Module organization: start grouped, split when complex
